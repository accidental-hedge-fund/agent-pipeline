## Why

Cutting a release today requires running a multi-step manual checklist — bump two `package.json`s, run `build.mjs` to regenerate the `plugin/` mirror, edit four distinct spots in `ROADMAP.md`, run `npm run ci`, then open the PR — and any missed step (especially the mirror regen or a ROADMAP table row) lands in CI or review failure. A `pipeline release` sub-command makes the repeatable mechanics one reviewed step, leaving the maintainer only the version decision, the review, and the merge.

## What Changes

- **New CLI sub-command**: `pipeline release <version>` where `<version>` is an explicit semver (`X.Y.Z`) or an alias (`major`, `minor`, `patch`) that is expanded against the current package version. Runs without an issue number; sits alongside `init`, `doctor`, and `cleanup` as a no-issue-number mode.
- **Version bump**: Updates `version` in both root `package.json` and `core/package.json` to the resolved version.
- **Mirror regen**: Runs `node scripts/build.mjs` after the bump so the `plugin/` mirror reflects the new version string before the PR is opened.
- **ROADMAP.md update (scaffolded)**: Edits four locations in `ROADMAP.md` atomically — marks the release-plan table row `✅ shipped`, prepends a new Shipped block (with scaffolded issue/PR rows derived from `git log <last-tag>..HEAD`), updates the intro "shipped chain" sentence, and adds a detail subsection — then opens the result in `$EDITOR` (or prints the diff under `--dry-run`) for the maintainer to edit before the PR is created.
- **Release PR creation**: Opens a PR titled `release: X.Y.Z — <theme>` after the maintainer confirms the ROADMAP diff. PR body summarizes the included issues/PRs since the last release tag.
- **`--dry-run` flag**: Prints the proposed version bump, ROADMAP diff, and PR body to stdout without writing any files or opening a PR.
- **CI gate**: Runs `npm run ci` (or asserts it passes) before the PR is opened. A failing CI gate aborts with a clear error.

## Capabilities

### New Capabilities
- `release-sub-command`: The `pipeline release` no-issue-number sub-command — argument parsing, version resolution (alias expansion + semver validation), pre-PR CI gate, version-bump + mirror-regen mechanics, ROADMAP scaffolding, PR creation, and `--dry-run` output.

### Modified Capabilities

## Impact

- `core/scripts/pipeline.ts` — new `isRelease` / `isReleaseCommand` dispatch branch; updated `.argument()` help text to list `release` as a recognized subcommand.
- `core/scripts/stages/release.ts` (new) — all release mechanics: version resolution, file writes, `build.mjs` invocation, `ROADMAP.md` edits, `npm run ci` invocation, PR creation.
- Root `package.json` and `core/package.json` — written at release time (by the command, not at design time).
- `ROADMAP.md` — four locations patched by the command at release time.
- `plugin/` mirror — regenerated by `build.mjs` inside the release command before PR open.
- `core/test/release.test.ts` (new) — unit tests for version resolution, ROADMAP diff logic, and dry-run output via `deps` injection seam.
- No changes to the pipeline state-machine, any review stage, or any existing stage handler.

## Acceptance Criteria

- [ ] `pipeline release 1.6.0` bumps `version` in both `package.json` files to `1.6.0`, regenerates the `plugin/` mirror, scaffolds + edits `ROADMAP.md`, runs `npm run ci`, and opens a PR — all without touching any issue number or pipeline stage label.
- [ ] `pipeline release minor` expands the alias against the current `core/package.json` version and behaves identically to `pipeline release <resolved-X.Y.Z>`.
- [ ] `pipeline release --dry-run 1.6.0` prints the resolved version, the ROADMAP diff, and the PR body to stdout and exits 0 without writing any file or opening a PR.
- [ ] An invalid version argument (e.g., `pipeline release foo`) exits with a non-zero code and an error message before touching any file.
- [ ] A failing `npm run ci` aborts the command before any file is written or any PR is opened.
- [ ] The opened PR title matches `release: X.Y.Z — <theme>` and its body lists the issues/PRs merged since the last release tag.
- [ ] The command refuses to run if invoked with an issue number (e.g., `pipeline release 42` where 42 is numeric) and prints a clear error distinguishing it from the `release <version>` form.
- [ ] `npm run ci` passes in the repo after the version bump + mirror regen.
